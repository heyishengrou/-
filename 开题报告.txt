一、课题来源及研究的目的和意义
1.1 课题来源
本课题来源于**教育数据挖掘（Educational Data Mining, EDM）与人工智能教育（AI in Education, AIED）的前沿交叉领域。随着在线学习平台（MOOCs、智能题库）的普及，积累了大量的学生交互数据。本课题旨在利用图神经网络（Graph Neural Network, GNN）**这一先进的深度学习技术，解决传统知识追踪模型在处理复杂知识结构和提升可解释性方面的不足。

主要研究思路源自 ICLR 2019 论文 GRAPH-BASED KNOWLEDGE TRACING (GKT) [1]，该研究首次将 GNN 引入知识追踪（Knowledge Tracing, KT）任务，开创了结构化知识建模的新范式。

1.2 研究目的
本研究的目的在于建立一个能够准确、动态、结构化地追踪学生知识掌握状态的模型，具体目标包括：

提高预测精度： 克服传统模型（如 BKT、DKT）中知识点相互独立或缺乏结构化信息的缺陷，利用知识图谱（知识概念间的关系）作为归纳偏置（Relational Inductive Bias）来提升学生未来答题表现的预测精度。

实现动态追踪： 针对每个学生，构建并实时更新其对知识图谱中各知识概念的高维熟练度向量（Proficiency Vector），实现精细化的学生能力画像。

增强模型可解释性： 利用 GNN 消息传递机制的结构化特性，实现对模型预测结果和学生知识状态变化的可解释性分析与可视化诊断。

1.3 研究意义
（1）学术意义：

本研究是对 GNN 在时序教育数据应用领域的深化探索。通过在公开数据集上验证 GKT 模型相对于经典 KT 模型（如 BKT、DKT）的性能提升，论证了知识结构在学习过程中的重要性。研究结果将为教育数据挖掘领域提供一种新的、高效且可解释的结构化建模方法，推动知识追踪模型从序列依赖向结构依赖演进。

（2）应用意义：

本课题的成果将直接服务于智能教育系统（ITS）。精确的学生能力画像是实现自适应学习系统（Adaptive Learning System）和个性化教学推荐的基础。具体体现在：

教学诊断： 教师和学生可直观了解知识漏洞的根源，并基于图结构分析知识点之间的关联性。

资源推荐： 系统能够根据学生当前熟练度和知识图谱中的路径依赖，推荐最需要巩固或预习的习题或知识点，实现定制化学习路径。

二、国内外研究现状
2.1 知识追踪模型
贝叶斯知识追踪（BKT） [3] 是最早、最经典的 KT 模型，基于隐马尔可夫模型（HMM）。它假设学生知识状态是二元的（掌握/未掌握），并使用四个概率参数（初始概率、习得概率、猜对概率、失误概率）进行推理。现状分析： BKT 模型参数少，可解释性强，但假设所有知识点相互独立，且知识状态表达能力弱（二元），无法处理知识点之间的复杂依赖关系，导致在大规模、复杂数据集上预测精度受限。
2.2 深度学习知识追踪模型（DKT）
深度知识追踪（DKT） [2] 是第一个将深度学习（循环神经网络 RNN/LSTM）引入 KT 的模型。它将学生的答题序列视为时间序列数据，利用 LSTM 强大的序列建模能力来捕捉学生知识状态随时间的变化。
现状分析： DKT 及其后续变体（如 DKVMN [11]）显著提升了预测精度。然而，它们仅能隐式地从序列中学习知识点间的关系，没有显式地将学科知识的结构化图谱信息作为先验知识嵌入模型，因此其学习到的知识关系缺乏教育学的可解释性和可靠性。
2.3 图基知识追踪模型（GKT）及图模型发展
为了解决深度学习模型缺乏结构化先验知识的问题，研究人员开始引入图模型：
GKT (Graph-based Knowledge Tracing) [1]：
核心突破： 将知识概念定义为图的节点，将知识点间的关系定义为图的边。利用 **GNN（如 GCN）**的消息传递机制，实现了知识状态的关联更新。当学生练习一个知识点 $A$ 时，其相关知识点 $B$ 的熟练度也会通过图上的边权重同步更新。
优点： 显式建模知识结构，提高了预测精度和模型可解释性。
后续发展：R-GCN： 针对知识图谱中存在的多种关系类型（如先决条件、相似性、包含关系），研究人员引入关系图卷积网络（R-GCN） [10] 来处理异构知识图谱 [13]。
自动化图学习（AGKT）： 由于知识图谱的构建成本高，自动化图基知识追踪（AGKT） [7] 等模型被提出，尝试在训练过程中自适应地学习知识概念之间的隐式关系矩阵，进一步解放了对人工图谱的依赖。
2.4 国内知识图谱与能力画像研究（参考辅助文献）
国内研究者已将知识图谱和知识追踪集成到自适应学习系统和学生能力画像设计中：
能力画像设计： 方志伟等研究了如何基于知识图谱来设计智能化的学生能力画像 [5]，强调了知识图谱在知识诊断、路径规划中的核心作用。
系统构建： 段夕瑜等研究了基于知识图谱和知识追踪的自适应学习系统的构建 [6]，表明 GKT 类模型是系统核心计算引擎的理想选择。
教学应用： 付璐等则将知识图谱应用于专业课程的“教学评”一体化研究 [12]，证明了图结构在教育实践中的广阔前景。
本课题将立足于 GKT 模型，并在实现过程中引入 AGKT 的思想和知识图谱工程化的经验，旨在提供一个高精度、高可解释性的学生能力建模方案。
3.1 研究内容
三、主要研究内容
本课题将围绕 GKT 核心模型和学生能力画像的工程化实现展开，主要包括以下四个方面的内容：
知识图谱构建与表示：研究现有公开教育数据集（如 ASSISTments）的知识概念（Skill ID）及其相互关系。采用数据驱动的共现分析方法作为基准，构建知识概念间的邻接矩阵 $\tilde{A}$，并将图谱结构数据化。可选方案： 调研并初步尝试使用 Neo4j 等图数据库对构建的知识图谱进行持久化管理，以支持复杂的查询和未来的系统扩展。
GKT 模型设计与实现：

基于 PyTorch 深度学习框架和 DGL/PyTorch Geometric 图神经网络库，复现 GKT 核心模型结构。

将知识追踪任务重新定义为 GNN 中的时序节点分类问题，实现知识状态初始化、消息传递（GCN/R-GCN）、知识状态更新三大模块。

关键机制： 实现 GKT 的**“读-写”**机制，即 GNN 通过聚合邻居信息来更新被练习知识点的状态，并同时影响其相邻知识点的状态。
学生能力动态追踪与预测：
将学生历史答题记录转换为 GKT 模型所需的序列输入，实时计算并存储每个时间步 $t$ 的学生知识状态向量 $h^{(t)}$。基于最新的知识状态 $h^{(T)}$，对学生在下一道习题 $q_{T+1}$ 上的答对概率进行精确预测。
模型可解释性分析与可视化：

借鉴 GKT 原文中的知识状态变化描绘 (Knowledge State Change Depiction) [1] 方法，对学生在练习过程中关键知识点的熟练度变化轨迹进行量化分析。

利用可视化工具（如 ECharts 或 D3.js）将知识图谱的结构和学生的动态熟练度（映射为节点颜色、大小）进行集成展示，实现直观的学生能力诊断画像。
3.2 研究方法    
技术路线
本课题采用**“数据准备-模型构建-实验验证-应用展示”**的工程化技术路线。

技术栈： | 模块 | 核心技术 | 目的 | | :--- | :--- | :--- | | 数据处理 | Python, Pandas | 数据清洗，序列化，习题-概念映射。 | | 图结构管理 | Python NetworkX 或 Neo4j (可选) | 知识图谱的逻辑构建、存储和查询。 | | 模型实现 | PyTorch | 深度学习框架。 | | GNN 核心 | DGL / PyTorch Geometric | GNN 消息传递机制的快速实现。 | | 性能验证 | Scikit-learn, CUDA/GPU | 实验评估、加速计算。 |
实施步骤数据采集与预处理 (第 1-4 周)：选取 ASSISTments 等公开教育数据集，完成数据清洗、匿名化处理和习题-概念映射确认。将学生的答题记录转换为 GKT 所需的**（习题 ID，答题结果）**序列。知识图谱构建与数据集成 (第 5-8 周)：根据选定数据集的知识点 ID，采用共现频率法构建知识概念间的邻接矩阵 $\tilde{A}$。将 $\tilde{A}$ 矩阵集成到 PyTorch 张量或 DGL/PyG 的图对象中。（可选） 完成 Neo4j 数据库的搭建，将知识图谱结构导入 Neo4j，并编写 Python 接口程序。GKT 模型核心实现 (第 9-12 周)：实现 GKT 模型的主体架构：嵌入层、GCN/GAT 消息传递层、时间依赖更新机制（序列模型/GRU）、知识状态更新层和最终预测层。完成模型训练、验证和测试的数据划分及加载器（DataLoader）的实现。实验验证与结果分析 (第 13-16 周)：在选定数据集上进行模型训练，以 AUC 和 Accuracy 作为主要评估指标。将 GKT 性能与基准模型（BKT, DKT）进行对比，验证 GKT 模型的优越性。进行消融实验，分析知识图谱结构对模型性能的贡献。可解释性分析与可视化展示 (第 17-20 周)：实现知识状态变化描绘的可视化模块，用于展示学生对一组相关知识点的熟练度变化。完成基于 GKT 模型的学生能力画像的可视化界面。完成毕业论文的撰写、修改和定稿。
